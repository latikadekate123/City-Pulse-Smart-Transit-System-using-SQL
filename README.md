# City-Pulse-Smart-Transit-System-using-SQL

This project is a PostgreSQL-based system I built to simulate how a city's public transport works behind the scenes. This has been done by tracking passengers, booking seats, and spotting overcrowded routes using real-time-style data.

## What It Does

- **Track seat availability**: See how many seats are left for each bus or metro route at different times.
- **Detect overloaded routes**: Identify when and where public transport is overused.
- **Analyze traffic by zone**: Understand which areas have the most passenger activity.
- **Let users book seats**: A simple booking system where 5 users can reserve seats in advance.

## SQL Code Lines 

**tablesCity.sql** : SQL script that I used to create all the tables (routes, zones, users, bookings, logs).

**dataCity.sql** : Inserted some initial manual data like routes, zones, users, and sample logs.

**queriesCity.sql** : Useful queries to check seat availability, bookings, and route load.
